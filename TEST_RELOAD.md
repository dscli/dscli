# 🔄 dscli --reload 功能测试指南

## 测试目标
验证 `dscli chat --reload` 功能能够正确传递所有命令行参数。

## 已实现的功能

### 1. 命令行参数传递
- ✅ 重载时传递所有原始命令行参数
- ✅ 自动添加 `--reload` 标志（如果不存在）
- ✅ 使用绝对路径避免递归

### 2. 参数验证
- ✅ 必须提供 `confirm=yes` 参数
- ✅ 检查是否是重载进程

### 3. 上下文传递
- ✅ 命令行参数存储在上下文中
- ✅ 重载时读取上下文中的参数

## 测试步骤

### 步骤1：启动 dscli chat 会话
```bash
# 使用多个参数启动会话
echo "测试重载功能" | dscli chat \
  --model deepseek-chat \
  --mode org \
  --no-color \
  --log-level debug
```

### 步骤2：让 AI 调用重载工具
在会话中，让 AI 调用 `dscli_chat_reload` 工具：
```json
{
  "confirm": "yes"
}
```

### 步骤3：验证重载行为
观察以下行为：
1. 显示 `"🔄 检测到重载命令，正在重启进程..."`
2. 进程正确重启
3. 没有无限递归
4. 所有原始参数保持：
   - `--model deepseek-chat`
   - `--mode org`
   - `--no-color`
   - `--log-level debug`
   - `--reload`（自动添加）

### 步骤4：验证重载后的功能
1. 对话历史应该保持
2. 未完成的工具调用应该继续
3. 输出格式应该保持（org 模式，无颜色）

## 故障排除

### 问题1：无限递归
**症状**：进程不断重启
**原因**：`--reload` 标志被重复添加
**解决**：已修复 - 检查是否已有 `--reload` 标志

### 问题2：参数丢失
**症状**：重载后参数改变
**原因**：命令行参数未正确传递
**解决**：已修复 - 使用 `os.Args` 传递所有参数

### 问题3：找不到 dscli 命令
**症状**：`找不到 dscli 命令`
**原因**：`$PATH` 环境变量问题
**解决**：使用 `exec.LookPath` 获取绝对路径
