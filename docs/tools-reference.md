# dscli 工具参考手册

## 概述

dscli 提供了丰富的工具集，帮助AI助手与开发环境进行交互。所有工具都在项目根目录下执行，支持相对路径和绝对路径。

## 工具分类

### 1. 文件操作工具

#### read_file
**描述**: 读取项目内指定文件的内容，返回文件内容和元数据信息（大小、权限、修改时间等）

**参数**:
- `path` (必需): 文件路径，如 `main.go`

**示例**:
```json
{
  "path": "README.md"
}
```

#### write_file
**描述**: 将内容写入文件，如果文件不存在则创建，如果存在则覆盖。支持创建目录结构。

**参数**:
- `path` (必需): 文件路径，如 `main.go`
- `content` (必需): 要写入的内容

**示例**:
```json
{
  "path": "test.txt",
  "content": "Hello, World!"
}
```

#### read_file_with_line_range
**描述**: 读取文件指定行范围的内容，输出格式与awk完全兼容。支持从某行到末尾的读取。

**参数**:
- `path` (必需): 文件路径
- `start_line` (可选): 起始行号（从1开始），默认1
- `end_line` (可选): 结束行号，默认到文件末尾

**示例**:
```json
{
  "path": "main.go",
  "start_line": 10,
  "end_line": "20"
}
```

#### write_file_with_line_range
**描述**: 写入文件指定行范围的内容，支持替换、插入和删除操作。支持新建文件和清空文件。

**参数**:
- `path` (必需): 文件路径
- `content` (必需): 要写入的内容，可以为空字符串表示删除
- `start_line` (可选): 起始行号，默认1
- `end_line` (可选): 结束行号，默认到文件末尾

**示例**:
```json
{
  "path": "file.txt",
  "start_line": 5,
  "end_line": 10,
  "content": "新内容"
}
```

#### search_files
**描述**: 在项目目录中搜索文件，支持文件名模式匹配（如`*.go`）和文件内容搜索。自动排除`.git`目录。

**参数**:
- `pattern` (可选): 文件名模式，如 `*.go`，为空则匹配所有文件
- `content` (可选): 要搜索的内容（如果提供则搜索文件内容）

**示例**:
```json
{
  "pattern": "*.go",
  "content": "TODO"
}
```

#### search_file_with_pattern
**描述**: 搜索文件中匹配指定模式的行，并显示上下文内容。支持大小写敏感搜索和匹配数限制。

**参数**:
- `path` (必需): 文件路径
- `pattern` (必需): 搜索模式（字符串包含匹配）
- `context_lines` (可选): 上下文行数（前后各N行），默认5
- `case_sensitive` (可选): 是否区分大小写，默认false
- `max_matches` (可选): 最大匹配数，默认无限制

**示例**:
```json
{
  "path": "main.go",
  "pattern": "TODO",
  "context_lines": "3"
}
```

### 2. Git操作工具

#### git_add
**描述**: 将指定文件添加到Git暂存区，支持多个文件用空格分隔。

**参数**:
- `path` (必需): 文件路径（相对于项目根目录），多个文件用空格分隔

**示例**:
```json
{
  "path": "main.go tools.go"
}
```

#### git_commit
**描述**: 提交暂存区更改，需要提供提交信息。注意：不要在options中包含`-m`或`--message`参数。

**参数**:
- `message` (必需): 提交信息
- `options` (可选): 其他git commit选项，例如：`-a`（提交所有更改）、`--amend`（修改上次提交）、`--no-edit`（使用原提交信息）、`--allow-empty`（允许空提交）。多个选项用空格分隔，例如：`-a --amend --no-edit`

**示例**:
```json
{
  "message": "修复bug",
  "options": "-a"
}
```

#### git_log
**描述**: 查看Git提交历史，支持限制显示数量。

**参数**:
- `max_count` (可选): 最大显示数量，默认"10"

**示例**:
```json
{
  "max_count": "5"
}
```

#### git_diff
**描述**: 查看工作区与暂存区或HEAD的差异，支持指定文件。

**参数**:
- `path` (可选): 文件路径（相对于项目根目录），多个文件用空格分隔

**示例**:
```json
{
  "path": "main.go"
}
```

#### git_status
**描述**: 查看Git仓库状态，显示已修改、已暂存和未跟踪的文件。

**参数**: 无

**示例**:
```json
{}
```

#### git_push
**描述**: 推送本地提交到远程仓库，支持各种git push选项。

**参数**:
- `options` (可选): 选项，例如：`--force-with-lease`，多个选项用空格分隔，例如：`origin main --force`，可为空

**示例**:
```json
{
  "options": "origin main"
}
```

### 3. 脚本执行工具

#### shell
**描述**: 在项目根目录执行Shell脚本，支持shebang指定解释器。返回格式化的执行结果和统计信息。

**参数**:
- `script` (必需): 要执行的Shell脚本内容

**示例**:
```json
{
  "script": "echo \"Hello\" && ls -la"
}
```

#### python
**描述**: 在项目根目录执行Python脚本，自动添加python3 shebang（如果未提供）。返回格式化的执行结果和统计信息。

**参数**:
- `script` (必需): 要执行的Python脚本内容

**示例**:
```json
{
  "script": "print(\"Hello\")\nimport os\nprint(os.getcwd())"
}
```

### 4. 数据库工具

#### sqlite
**描述**: 执行SQLite数据库查询和操作，脚本内容为SQL语句。支持`.schema`等sqlite3命令。

**参数**:
- `script` (必需): sqlite SQL脚本内容

**示例**:
```json
{
  "script": ".schema messages"
}
```

## 使用建议

1. **文件路径**: 使用相对路径时，相对于项目根目录
2. **错误处理**: 所有工具都提供详细的错误信息和执行统计
4. **工具选择**: 根据任务类型选择合适的工具，如文件操作使用文件工具，脚本执行使用shell或python工具
5. **安全注意**: 谨慎使用脚本执行工具，避免破坏性操作
