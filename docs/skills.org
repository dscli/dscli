* 概述
** 什么是 Skills 系统？
   Skills 系统是 dscli 的一个核心特性，它允许用户定义、管理和应用最佳
   实践规则（称为"技能"）。这些技能会在与 DeepSeek AI 对话时自动注入到
   系统提示中，从而影响 AI 的行为和决策。

** 核心理念
   - **动态组合**：不同于传统的静态系统提示，Skills 系统支持按项目动态
     组合技能
   - **用户驱动**：用户可以根据自己的需求创建和管理技能
   - **自我优化**：AI 可以通过工具调用管理自己的技能库
   - **上下文感知**：技能可以根据项目类型和上下文条件化启用

     -- 技能主表
     CREATE TABLE skills (
         id INTEGER PRIMARY KEY AUTOINCREMENT,
         name TEXT NOT NULL UNIQUE,           -- 技能名称
         description TEXT NOT NULL,           -- 技能描述
         content TEXT NOT NULL,               -- 技能内容/规则
         category TEXT,                       -- 分类：git, file_ops, code_quality 等
         priority INTEGER DEFAULT 50,         -- 优先级 (0-100)
         is_global BOOLEAN DEFAULT 0,         -- 是否全局技能
         usage_count INTEGER DEFAULT 0,       -- 使用次数统计
         created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
         updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
     );

     -- 项目技能关联表
     CREATE TABLE project_skills (
         project_hash TEXT NOT NULL,          -- 项目路径哈希
         skill_id INTEGER NOT NULL,
         is_enabled BOOLEAN DEFAULT 1,        -- 在该项目中是否启用
         enabled_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
         last_used TIMESTAMP,                 -- 最后使用时间
         PRIMARY KEY (project_hash, skill_id),
         FOREIGN KEY (skill_id) REFERENCES skills(id) ON DELETE CASCADE
     );

* 设计哲学
** Skills 与系统提示的区别
   | 维度     | 传统系统提示         | Skills 系统                  |
   |----------+----------------------+------------------------------|
   | 存储方式 | 硬编码在代码中       | SQLite 数据库表              |
   | 修改方式 | 需要改代码、重新编译 | 运行时 CRUD 操作             |
   | 作用范围 | 全局统一             | 可按项目配置                 |
   | 个性化   | 无                   | 每个项目不同技能集           |
   | 条件启用 | 很难实现             | 基于项目类型、文件存在等条件 |
   #+TBLFM: 

** 本质区别
   - **系统提示**：定义 AI 的**能力**（你能做什么）
   - **Skills**：定义 AI 的**行为偏好**（你该如何做）
** 类比
   - **系统提示** = 操作系统的内核（提供基础能力）
   - **Skills** = 用户安装的应用程序（定义如何使用这些能力）
