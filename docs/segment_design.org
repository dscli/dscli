** 概述
   :PROPERTIES:
   :CUSTOM_ID: 概述
   :END:

   段落系统是dscli的核心功能之一，用于管理和组织系统提示词。它取代了之
   前分散的template概念，统一使用"段落"这一概念，减少用户心智负担。

** 核心概念
   :PROPERTIES:
   :CUSTOM_ID: 核心概念
   :END:

*** 1. 段落（Segment）
    :PROPERTIES:
    :CUSTOM_ID: 段落segment
    :END:

- *定义*：一个独立的提示词片段，包含特定的指令、规则或上下文信息
- *特点*：

  - 支持模板变量（如={{.CurrentDate}}=、={{.ProjectName}}=等）
  - 可以按领域和模型分类
  - 支持启用/禁用状态
  - 有排序顺序控制显示位置

*** 2. 领域（Domain）
    :PROPERTIES:
    :CUSTOM_ID: 领域domain
    :END:

- *定义*：一个特定的知识或任务领域
- *示例*：编程开发、文档写作、数学研究、工业控制等
- *作用*：将段落按主题分类，支持不同项目的个性化配置

*** 3. 模型（Model）
    :PROPERTIES:
    :CUSTOM_ID: 模型model
    :END:

- *定义*：AI模型类型
- *支持*：

  - =-1=：通用（所有模型）
  - =0=：Deepseek Chat
  - =1=：Deepseek Reasoner

** 数据库结构
   :PROPERTIES:
   :CUSTOM_ID: 数据库结构
   :END:

*** prompt\_segments 表
    :PROPERTIES:
    :CUSTOM_ID: prompt_segments-表
    :END:

#+BEGIN_SRC sql
    CREATE TABLE prompt_segments (
        id INTEGER PRIMARY KEY AUTOINCREMENT,
        domain_id INTEGER NOT NULL,          -- 领域ID
        model_id INTEGER NOT NULL DEFAULT -1, -- 模型ID
        name TEXT NOT NULL,                   -- 段落名称
        content TEXT NOT NULL,                -- 段落内容（支持模板语法）
        sort_order INTEGER NOT NULL DEFAULT 0, -- 排序顺序
        enabled BOOLEAN NOT NULL DEFAULT true, -- 是否启用
        created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
        updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
        FOREIGN KEY (domain_id) REFERENCES domains(id) ON DELETE CASCADE
    );
#+END_SRC

*** domains 表
    :PROPERTIES:
    :CUSTOM_ID: domains-表
    :END:

#+BEGIN_SRC sql
    CREATE TABLE domains (
        id INTEGER PRIMARY KEY AUTOINCREMENT,
        name TEXT NOT NULL,                   -- 领域名称
        description TEXT,                     -- 领域描述
        created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
    );
#+END_SRC

*** project\_domains 表
    :PROPERTIES:
    :CUSTOM_ID: project_domains-表
    :END:

#+BEGIN_SRC sql
    CREATE TABLE project_domains (
        id INTEGER PRIMARY KEY AUTOINCREMENT,
        project_root TEXT NOT NULL,           -- 项目根目录
        domain_id INTEGER NOT NULL,           -- 领域ID
        created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
        FOREIGN KEY (domain_id) REFERENCES domains(id) ON DELETE CASCADE,
        UNIQUE(project_root)
    );
#+END_SRC

** 段落渲染流程
   :PROPERTIES:
   :CUSTOM_ID: 段落渲染流程
   :END:

1. *获取当前项目领域*：根据项目根目录查找对应的领域
2. *获取当前模型*：根据用户选择的模型确定模型ID
3. *加载段落*：从数据库加载对应领域和模型的段落
4. *渲染模板*：将模板变量替换为实际值
5. *拼接段落*：按排序顺序拼接所有段落
6. *生成系统提示词*：将拼接后的内容作为系统提示词

** 默认段落
   :PROPERTIES:
   :CUSTOM_ID: 默认段落
   :END:

系统初始化时会自动创建以下默认段落：

*** 系统级段落（domain\_id=0）
    :PROPERTIES:
    :CUSTOM_ID: 系统级段落domain_id0
    :END:

1. *Deepseek Chat 模板*（model\_id=0, sort\_order=10）

   - 包含完整的编程助手指令
   - 支持所有模板变量

2. *Deepseek Reasoner 模板*（model\_id=1, sort\_order=20）

   - 包含深入思考者的指令
   - 专注于逻辑推理和深度分析

*** 编程领域段落（domain\_id=1）
    :PROPERTIES:
    :CUSTOM_ID: 编程领域段落domain_id1
    :END:

1. *基本原则*（model\_id=-1, sort\_order=10）

   - "你是一个专业的编程助手。"

2. *工作流程*（model\_id=-1, sort\_order=20）

   - 包含4步工作流程

3. *代码质量要求*（model\_id=-1, sort\_order=30）

   - 包含4条代码质量原则

    :END:

    :CUSTOM_ID: 灵活配置
    :END:

- 支持按领域、模型、项目进行个性化配置
- 段落可以启用/禁用，方便调试

*** 3. 向后兼容
    :PROPERTIES:
    :CUSTOM_ID: 向后兼容
    :END:

- 保持与现有系统的兼容性
- 默认段落确保基本功能正常工作

*** 4. 易于扩展
    :PROPERTIES:
    :CUSTOM_ID: 易于扩展
    :END:

- 支持添加新的领域和模型

** 最佳实践
   :PROPERTIES:
   :CUSTOM_ID: 最佳实践
   :END:

*** 1. 段落设计
    :PROPERTIES:
    :CUSTOM_ID: 段落设计
    :END:

- 每个段落应该专注于一个特定的主题
- 保持段落简洁，避免过长
- 使用模板变量增加灵活性

*** 2. 领域划分
    :PROPERTIES:
    :CUSTOM_ID: 领域划分
    :END:

- 根据项目类型选择合适的领域
- 可以为特定项目创建自定义领域

*** 3. 模型适配
    :PROPERTIES:
    :CUSTOM_ID: 模型适配
    :END:

- 为不同模型设计不同的段落内容
- Chat模型：注重实用性和操作性
- Reasoner模型：注重深度思考和逻辑推理

*** 4. 版本控制
    :PROPERTIES:
    :CUSTOM_ID: 版本控制
    :END:

- 重要的段落变更应该记录在Git中
- 可以使用数据库备份功能

** 故障排除
   :PROPERTIES:
   :CUSTOM_ID: 故障排除
   :END:

*** 1. 段落不生效
    :PROPERTIES:

- 检查领域和模型是否匹配

    :CUSTOM_ID: 模板变量未替换
    :END:

- 检查变量名称是否正确
- 检查变量是否在可用变量列表中
- 检查模板语法是否正确

*** 3. 段落冲突
    :PROPERTIES:
    :CUSTOM_ID: 段落冲突
    :END:

- 检查是否有重复的段落
- 检查排序顺序是否导致覆盖
