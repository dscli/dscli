### 基于大文件编辑体验的改进任务

**问题分析**：在编辑大文件（如tools.go）时遇到以下问题：
1. 大文件编辑困难：超过1000行的文件使用 `write_file_with_line_range` 容易出错
2. 重复内容问题：多次编辑后出现重复的函数定义
3. 语法错误难以定位：Go编译器报告的行号有时不准确
4. 缺少组合操作：没有"查找并替换"、"查找并插入"等常用操作

**改进目标**：提供更安全、更高效的大文件编辑工具集

#### 文件短期改进三件套（高优先级）

**设计理念**：基于实际编辑大文件的痛点，提供三个核心工具，显著提升编辑效率和安全性。

##### 1. `file_search_replace` - 搜索替换工具
- [ ] **任务1.14：实现 file_search_replace 工具**
  - 文件：`file.go`
  - 描述：在文件中搜索并替换内容，解决重复内容问题
  - 功能特性：

    - ⬜ **替换选项**：支持全局替换、单次替换、确认后替换
    - ⬜ **预览功能**：显示替换前后的差异预览
    - ⬜ **统计信息**：显示匹配数量和替换数量

##### 2. `file_insert_with_pattern` - 模式插入工具
- [ ] **任务1.15：实现 file_insert_with_pattern 工具**
  - 文件：`file.go`
  - 描述：基于模式匹配插入内容，避免精确计算行号
  - 功能特性：
    - ⬜ **插入位置**：支持 before（之前）、after（之后）、replace（替换）
    - ⬜ **模式匹配**：支持字符串包含匹配和正则表达式
    - ⬜ **多行匹配**：支持多行模式匹配
    - ⬜ **精确控制**：支持匹配第N次出现的位置
    - ⬜ **上下文显示**：显示匹配位置的上下文内容
  - 参数设计：
    - `path`：文件路径（必需）
    - `pattern`：搜索模式（必需）
    - `content`：要插入的内容（必需）
    - `position`：插入位置（必需，"before"、"after"、"replace"）
    - `pattern_type`：模式类型（可选，"contains" 或 "regex"，默认 "contains"）
    - `occurrence`：匹配第几次出现（可选，默认 1）
    - `case_sensitive`：是否区分大小写（可选，默认 false）
  - 相关函数：`handleFileInsertWithPattern`
  - 难度：中等
  - 优先级：高
  - **解决痛点**：大文件编辑时无需精确计算行号

##### 3. 备份策略说明
**设计决策**：不实现专门的 `file_backup` 工具，理由如下：
1. **已有替代方案**：Git 已经提供了完善的版本控制功能
2. **操作建议**：在重大修改前手动执行 `git add` 和 `git commit`
3. **最佳实践**：鼓励使用版本控制而不是临时备份
4. **简化设计**：减少工具数量，保持核心功能聚焦

**替代方案**：
- 在执行 `file_search_replace` 和 `file_insert_with_pattern` 前，可以提示用户先提交Git
- 可以提供Git操作快捷方式，如 `git_snapshot` 工具
- 可以在工具文档中强调Git备份的重要性

**实施建议**：

## 🏗️ 架构约束提醒
**重要**：所有新功能必须遵循以下原则：
1. **不创建子目录**：所有Go文件都在根目录下
2. **一对一测试**：每个新功能文件必须有对应的测试文件
3. **文件命名**：功能文件为 `xxx.go`，测试文件为 `xxx_test.go`
4. **包名一致**：所有文件使用相同的包名 `main`

## 📅 日期处理规范
**重要**：所有日期相关功能必须遵循：
1. **动态日期**：系统提示词中的日期必须动态生成
2. **文档日期**：文档中的日期必须反映实际更新时间
3. **测试日期**：测试中使用可预测的日期
4. **版本日期**：版本信息包含构建日期
